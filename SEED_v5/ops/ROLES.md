# ロール定義とシステム上の役割 (Roles & Responsibilities - BPO Model)

BPOセンター運営を想定した役割定義と、システム（Ops OS）上での担当範囲。

## 1. 問い合わせ者 (Requester)
- **主体**: 顧客企業の社員
- **役割**: 業務依頼を行う。
- **画面**: 受付フォーム

## 2. 事務センター (Ops Center)
### 2-1. 作業者 (Operator)
- **主体**: 現場オペレータ
- **役割**: 定型業務の遂行。
- **画面**: Opsコンソール (Inbox / Guide / Draft)
- **アクション**:
  - 標準プロセス（ガイド）に沿って処理。
  - 判断に迷うものは「SV承認待ち」へ回す。

### 2-2. 作業リーダー (Supervisor / SV)
- **主体**: チームリーダー
- **役割**: 現場の一次エスカレーション先。
- **画面**: Opsコンソール (Wait / Exception)
- **アクション**:
  - オペレータからの相談を即時判断（承認/却下）。
  - **「自分では裁けない例外」**や**「リスク案件」**をPMへ報告する。

### 2-3. プロジェクトマネージャー (PM)
- **主体**: 運用管理者
- **役割**: 顧客折衝、リスク管理、**ルールの進化（SEED進化）**。
- **画面**: Opsコンソール (Analytics / Process Mgmt - ※今後実装)
- **ワークフロー**:
  1. **例外・リスク検知**: SVから上がってきた「裁けない例外」を受け取る。
  2. **顧客協議**: 顧客企業へ報告し、対処指示を仰ぐ。
  3. **ルール策定**: 新しい業務ルール（マニュアル変更案）を作成する。
  4. **承認・反映**: 顧客の承認を得て、システム（ガイド/チェックリスト）を更新する。
  5. **周知**: SV・オペレータへ変更を周知する。

## 3. 顧客企業 (Client)
- **主体**: 委託元の担当者
- **役割**: 運用ルールの最終決定、例外の判断指示。
